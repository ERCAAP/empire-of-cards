# Venture: Tech App

## 1. Identity

Tech App fiziksel dükkan yerine ürün, kullanıcı akışı, backend ve support üzerinden oynar. Dünya yine diorama olarak temsil edilir: kullanıcı akışları, ticket balonları, app store rating paneli, developer masaları ve server node'ları görünür.

Fantasy:

> "Basit bir uygulamayı viral bir teknoloji şirketine çevirdim."

## 2. Starting Board

- Küçük startup ofisi
- 1 product workstation
- 1 backend node
- 1 developer
- 1 founder/operator
- Store rating: 4.0
- Düşük user base

## 9. Chain Scenarios

### Good Growth

ASO + Onboarding + Support -> rating yükselir -> organic installs -> churn düşer.

### Overgrowth Collapse

Influencer Launch + zayıf backend -> crash spike -> bad reviews -> store rating düşer -> installs rakibe kayar.

### Risk Debt

Fake Store Reviews + Cheap Cloud -> kısa rating boost -> platform audit veya outage -> reputation damage.

## 11. Deep Venture Scenarios

### 11.1 İyi Büyüme: Sağlıklı Product-Led Growth

`Onboarding Redesign`, `UX Designer`, `Support Hire` ve `ASO İyileştirme` birlikte çalışır. User stream yavaş ama temiz akar, support ticket balonları düşük kalır. Store rating artar ve organik install büyür.

### 11.2 Aşırı Büyüme Çöküşü: Influencer Launch Crash

`Influencer Launch` burst olarak büyük user wave üretir. Backend zayıfsa server node kızarır, crash balonları çıkar. Event: `Crash Spike`. Oyuncu hotfix sprint, premium cloud veya kampanyayı durdurma arasında seçim yapar.

### 11.3 Çalışan Krizi: Crunch Sprint Bedeli

Oyuncu `Crunch Sprint` ile feature velocity kazanır. Developer stress yükselir, ofiste gece ışıkları yanar. Birkaç tur sonra developer burnout event'i gelir. Zam/izin vermek velocity düşürür, zorlamak quit risk'i üretir.

### 11.4 Rakip Baskısı: Feature Copy

Rakip oyuncunun başarılı feature'ını kopyalar. Rival tarafında kırmızı feature billboard'u görünür, bazı gri kullanıcılar kırmızı stream'e akar. Oyuncu quality polish, pricing, community veya yeni feature hamlesiyle cevap verir.

### 11.5 Riskli Kısa Yol: Fake Reviews Audit

`Fake Store Reviews` rating'i kısa süre şişirir. Store panelinde yıldızlar glitch'li parlar. Platform audit event'i gelirse rating düşebilir, app geçici olarak görünürlük kaybedebilir. Temiz çözüm public roadmap apology + hotfix; riskli çözüm yeni fake push.
